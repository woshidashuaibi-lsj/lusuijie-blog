## 博客系统

my-blog 项目的博客系统基于 Next.js 的静态生成（Static Generation）机制实现，通过 `getStaticProps` 和 `getStaticPaths` 在构建时预渲染所有博客页面。系统从 `posts/` 目录读取 Markdown 文件，利用 `gray-matter` 解析元数据（如标题、日期、分类、标签等），并通过 `remark` 将 Markdown 内容转换为 HTML。所有博客文章按分类（如 life、tech）组织，支持自动提取封面、描述和阅读时间估算。

博客数据的读取和处理逻辑封装在 `src/lib/blog.ts` 中，提供了 `getAllPosts`、`getPostBySlug`、`markdownToHtml` 等核心函数。前端页面 `src/pages/blog/[slug].tsx` 使用 `getStaticPaths` 生成所有博客的静态路径，并通过 `getStaticProps` 获取对应博客的完整数据，确保页面在构建时即完成渲染，提升加载性能和 SEO 效果。

**中文(中文)**
- **功能路径**：`/blog/[slug]`
- **数据源**：`posts/` 目录下的 `.md` 文件
- **核心逻辑**：静态生成 + Markdown 解析 + 元数据提取

## 相册功能

相册功能用于组织和展示摄影作品，采用分类与网格布局相结合的方式提升浏览体验。照片数据在 `src/lib/photos.ts` 中以结构化对象形式定义，每张照片包含 ID、图片链接、分类、标题、拍摄时间、地点和标签等信息。照片按分类（如“公园·春”、“庭园·夏”）组织，支持分类侧边栏导航和全量照片展示。

前端页面 `src/pages/photo/index.tsx` 使用 `getStaticProps` 在构建时获取所有照片数据，包括按分类分组的照片（`photosByCategories`）、所有分类列表（`categories`）和全部照片（`allPhotos`）。页面通过 `PhotoGrid` 和 `PhotoCard` 组件以响应式网格布局渲染照片，并支持点击照片弹出模态框进行查看。分类切换通过侧边栏 `CategorySidebar` 实现，提升用户交互体验。

**中文(中文)**
- **功能路径**：`/photo`
- **数据源**：`src/lib/photos.ts` 中的 `photos` 数组
- **核心逻辑**：静态生成 + 分类分组 + 网格布局 + 模态查看

## 访客留言

访客留言功能基于 Giscus 第三方评论系统实现，将 GitHub Issues 作为后端存储。用户在前端提交留言后，数据通过 API 路由 `src/pages/api/guestbook.ts` 提交至 GitHub 仓库的 Issues 中，实现去中心化的留言管理。该方式无需独立数据库，利用 GitHub 的权限控制和通知机制，确保留言的安全性和可维护性。

前端组件 `GuestbookPage` 渲染留言界面，用户输入内容后通过 POST 请求发送至 `/api/guestbook`。后端 API 接收请求后，验证内容合法性，并使用预配置的 `GITHUB_TOKEN` 调用 GitHub API 创建新的 Issue，自动添加 `guestbook` 标签以便分类管理。成功后返回确认信息，实现完整的数据流闭环。

**中文(中文)**
- **功能路径**：`/guestbook`
- **数据流**：前端 → API 路由 → GitHub API → GitHub Issues
- **核心逻辑**：GitHub Issues 集成 + API 路由 + 安全验证

## 主题切换

主题切换功能通过 React 状态管理和 CSS 变量实现，支持用户在亮色与暗色主题之间切换。核心组件 `ThemeToggle` 位于 `src/components/ThemeToggle/index.tsx`，使用 `useEffect` 在客户端挂载时读取用户偏好（`localStorage` 或系统设置），并动态为 `document.documentElement` 添加 `dark` 类名，触发 CSS 变量切换。

切换动画利用现代浏览器的 View Transition API 实现平滑过渡效果。点击按钮时，通过 `startViewTransition` 创建视图过渡，在暗色到亮色或亮色到暗色切换时，分别执行圆形展开或收缩的 `clipPath` 动画，提升用户体验。若浏览器不支持 View Transition，则降级为直接切换类名。

全局主题样式定义在 `src/styles/globals.css` 中，通过 CSS 自定义属性（如 `--bg-color`、`--text-color`）定义亮/暗色值，确保主题一致性。

**中文(中文)**
- **触发方式**：点击导航栏主题按钮
- **状态管理**：`localStorage` + `useEffect`
- **样式机制**：CSS 类名 + CSS 变量
- **动画支持**：View Transition API
