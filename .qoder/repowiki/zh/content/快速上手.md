## 简介

本指南旨在帮助新贡献者快速在本地搭建 `my-blog` 项目的开发环境。项目基于 Next.js 框架，采用 TypeScript 和 Markdown 构建静态博客网站，并部署于阿里云 OSS。本文将逐步介绍从克隆仓库到构建生产版本的完整流程，确保开发者能够顺利参与项目开发。

## 开发环境搭建

### 系统要求

在开始之前，请确保您的系统满足以下最低要求：
- Node.js 18.x 或更高版本
- npm 8.x 或更高版本

### 安装步骤

1. **克隆仓库**

   打开终端，执行以下命令以克隆项目仓库：

   ```bash
   git clone <repository-url>
   cd my-blog
   ```

2. **安装依赖**

   使用 npm 安装项目所需的所有依赖包：

   ```bash
   npm install
   ```

   此命令会根据 `package.json` 文件自动下载并配置所有依赖项。

## 配置环境变量

项目使用 `.env.local` 文件来管理本地开发环境的配置。请按以下步骤进行配置：

1. 复制环境变量模板：

   ```bash
   cp .env.local.example .env.local
   ```

2. 编辑 `.env.local` 文件，填写阿里云 OSS 凭证：

   ```env
   NODE_ENV=development
   NEXT_PUBLIC_SITE_URL=http://localhost:3000
   ```

   注意：`.env.local.example` 文件不存在，可参考 `.env.production` 文件结构手动创建并配置。

## 创建新的博客文章

### 文件位置

新的博客文章应创建在 `posts/tech/` 或 `posts/life/` 目录下，文件扩展名为 `.md`。

### Front Matter 元数据

每篇 Markdown 文章需包含以下 front matter 元数据：

```yaml
---
title: "文章标题"
date: "YYYY-MM-DD"
description: "文章描述"
cover: "/images/cover.jpg"
tags: ["标签1", "标签2"]
---
```

### 编写内容

在 front matter 之后，使用标准 Markdown 语法撰写文章正文。支持代码块、表格、链接等格式。

示例文章结构见 `posts/tech/nextjs-blog.md`。

## 构建与预览生产版本

### 构建生产版本

运行以下命令构建生产环境的静态文件：

```bash
npm run build
```

构建完成后，静态文件将输出至 `out/` 目录。

### 预览生产版本

构建完成后，可通过以下命令启动生产版本服务器进行预览：

```bash
npm run start
```

访问 `http://localhost:3000` 即可查看构建后的页面效果。

### 一键构建并部署

如需同时构建并部署到阿里云 OSS，可运行：

```bash
npm run deploy
```

该命令会先执行 `export` 导出静态文件，再调用 `scripts/upload-oss.js` 脚本上传至 OSS。

## 常见问题与解决方案

### 依赖安装失败

**问题**：`npm install` 报错或卡住。

**解决方案**：
- 检查网络连接，尝试使用国内镜像源（如 `npm config set registry https://registry.npmmirror.com`）
- 清除 npm 缓存：`npm cache clean --force`
- 删除 `node_modules` 和 `package-lock.json` 后重新安装

### 端口冲突

**问题**：开发服务器启动时报错“端口已被占用”。

**解决方案**：
- 更改启动端口：`npm run dev -- -p 3001`
- 查找并终止占用端口的进程：`lsof -i :3000`，然后 `kill -9 <PID>`

### 样式不生效

**问题**：CSS 样式未正确加载。

**解决方案**：
- 确保使用 CSS Modules 语法（如 `import styles from './Component.module.css'`）
- 检查文件路径是否正确
- 重启开发服务器

### 部署后 404 错误

**问题**：部署后访问网站返回 404。

**解决方案**：
- 检查阿里云 OSS 是否已开启“静态网站托管”
- 确认“默认首页”设置为 `index.html`
- 验证域名解析和备案状态
