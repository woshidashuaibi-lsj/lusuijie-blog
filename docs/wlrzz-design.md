## 四、得分算法

```typescript
// 每个选项对应 { 人物ID: 分值 }
type OptionWeight = Record<string, number>;

// 题目数据结构
interface Question {
  id: number;
  text: string;
  options: {
    label: string;        // "A" / "B" / "C" / "D"
    text: string;         // 选项文案
    weights: OptionWeight; // e.g. { FENGGE: 2, TIEBA_LAOYAO: 1 }
  }[];
}

// 计算结果：累加所有选项得分，取最高分人物
function calcResult(answers: OptionWeight[]): string {
  const total: Record<string, number> = {};
  for (const weight of answers) {
    for (const [id, score] of Object.entries(weight)) {
      total[id] = (total[id] ?? 0) + score;
    }
  }
  return Object.entries(total).sort((a, b) => b[1] - a[1])[0][0];
}
```

## 六、页面流程与交互设计

```
Landing（封面）
  ↓ 点击「开始测试」
Question 1 → 2 → ... → 15
  ↓ 选完最后一题，自动触发计算
Result（结果卡）
  ↓ 点击「再测一次」
回到 Landing
```

### 交互细节
- **进度条**：顶部显示当前题号（1/15），纯色线性进度条
- **选项动效**：选中后高亮（使用 `var(--secondary-color)`），0.2s 过渡
- **切题方式**：选中即自动切到下一题，无需点"下一题"按钮（仿股市 GBTI 体验）
- **结果动画**：结果卡从下方滑入，带轻微弹簧效果（`cubic-bezier(0.34, 1.56, 0.64, 1)`）
- **分享功能**：复制一段格式化文本到剪贴板（不依赖图片生成）

```

"干就完了，出了事再说"
整活力 80% | 话题度 90% | 争议性 90%
你是哪个？→ https://yourdomain.com/wlrzz
```

## 十、实施步骤（开发顺序）

1. **创建数据层**：编写 `data.ts`（题目 + 权重 + 人物档案）和 `calc.ts`（算法）
2. **搭建骨架**：创建 `WlrzzPage/index.tsx`，实现三段式 phase 状态机（landing → quiz → result）
3. **Landing 封面**：实现 `Landing` 组件，全屏视差背景 + 开始按钮
4. **题目流程**：实现 `QuizCard` + `ProgressBar`，选中即跳下一题
5. **结果页面**：实现 `ResultCard`，含人物配色、属性条、分享按钮
6. **路由接入**：创建 `pages/wlrzz/index.tsx` + 加入 Navigation
7. **移动端适配**：确保在 375px 宽度下体验完整

---

## 十一、扩展方向（可选）

- **分享图片生成**：用 `html2canvas` 截取结果卡，生成图片供下载/分享
- **人物详情页**：每个人物有独立的 `/wlrzz/[id]` 页面，深度介绍
- **排行榜**：接 Serverless 小 API，统计各人物测出比例
- **节日限定**：特殊版题目（如元旦限定「年度网络人物回顾」）
